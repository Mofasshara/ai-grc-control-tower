# AI System Specification

An AI System represents a governed AI capability subject to compliance controls.

## Mandatory Attributes
- System Name
- Business Purpose
- Intended Users
- Risk Classification (Low / Medium / High)
- AI Owner
- Deployment Environment
- Lifecycle Status

## Lifecycle States
- Draft
- Submitted
- Approved
- Active
- Retired

Purpose

Auditors think in systems, not code artifacts.
This document defines the compliance object model.

## Risk Classification Guidance

Low:
- Informational outputs only
- No impact on regulated decisions

Medium:
- Supports human decision-making
- Requires human approval

High:
- Influences regulated processes
- Requires enhanced controls and monitoring
